# CASE_STUDY_DEMO.FOUNDATION

## CASE_STUDY_DEMO.FOUNDATION.HEADLINE
What we implemented (Foundation)

### CASE_STUDY_DEMO.FOUNDATION.ENTITY_SHEET
1) Canonical “Entity Sheet”  
We created a structured dataset containing:
- NAP + hours (incl. holiday rule)
- Service taxonomy (repair/install/maintenance + equipment types)
- Cities + ZIPs served
- Trust signals (license, insurance, warranties, financing, certifications)

### CASE_STUDY_DEMO.FOUNDATION.SERVICE_GRAPH
2) Website “Service Graph”  
We built/updated:
- High-intent service pages (written for clarity + conversion)
- FAQs answering pricing/timeline/warranty/permit questions
- A Service Areas page that clearly states coverage boundaries

### CASE_STUDY_DEMO.FOUNDATION.SCHEMA
3) Structured data (Schema JSON-LD)  
We implemented Schema aligned to on-page content:
- LocalBusiness/HVACBusiness entity markup
- Service markup
- FAQPage markup (only where the FAQ appears on the page)

### CASE_STUDY_DEMO.FOUNDATION.GBP
4) Google Business Profile “safe edits”  
With owner permission:
- Categories/services alignment
- Description alignment with the service catalog
- Hours review + consistency check

### CASE_STUDY_DEMO.FOUNDATION.TRACKING
5) Tracking + baseline report
- Click-to-call tracking
- Form submit tracking
- Baseline snapshot + next-30-day action plan
